# RAG Bot 前端设计文档

## 概述

为 RAG 知识库问答系统实现单页聊天应用前端，采用单 HTML 文件方案，内嵌 CSS/JS，由 FastAPI 直接提供静态服务。

## 架构

**技术栈**: 原生 HTML/CSS/JavaScript，无构建工具依赖

**文件结构**:
```
app/web/
├── __init__.py
└── static/
    └── index.html    # 单文件前端（内嵌 CSS/JS）
```

**服务方式**: FastAPI 静态文件路由直接提供 `index.html`

## 功能模块

### 1. 对话问答
- 输入框支持多行文本（Shift+Enter 换行，Enter 发送）
- 显示用户问题和助手回答
- 回答下方显示引用文档列表
- 无知识库命中时显示提示信息

### 2. 流式响应
- 使用 Server-Sent Events (SSE) 实现逐字显示
- 发送请求到 `POST /chat/stream`
- 实时渲染 Markdown 格式回答
- 流式完成后显示引用文档

### 3. 会话历史
- 左侧显示历史会话列表
- 点击切换会话，加载历史消息
- 支持新建会话
- 显示会话标题（首条问题截断）

### 4. 文档上传
- 输入框旁的上传按钮
- 支持拖拽上传
- 支持格式：txt, md, pdf, doc, docx, xlsx, csv, json
- 上传后触发知识库更新

### 5. 系统状态
- 右上角显示服务健康状态指示灯
- 绿色 = 正常，红色 = 异常
- 定时轮询 `/health` 接口

## API 接口

### 现有接口
- `POST /chat` - 普通问答
- `GET /history/sessions` - 获取会话列表
- `GET /history/sessions/{id}/messages` - 获取消息记录
- `GET /health` - 健康检查

### 需新增接口
- `POST /chat/stream` - 流式问答（SSE）
- `POST /ingest/upload` - 上传文档到知识库

## UI 样式

**配色方案**:
- 主色：#10a37f（绿色，类似 ChatGPT）
- 背景：#f7f7f8（浅灰）
- 侧边栏：#202123（深色）
- 文字：#343541（深灰）

**响应式设计**:
- 桌面端：侧边栏 + 聊天区域
- 移动端：侧边栏可折叠

## 错误处理

- API 请求失败显示友好错误提示
- 网络断开时显示重连提示
- 上传文件格式不支持时显示提示

## 安全考虑

- 敏感问题过滤（已有后端支持）
- 文件上传大小限制（建议 10MB）
- XSS 防护（转义用户输入）
